### 跳转关键词
1. break：直接跳出整个循环，循环彻底结束，或者结束所在switch分支的执行
2. continue：跳过当前这一次，直接进入下一次循环，循环还继续跑；
注意 continue 一触发，迭代自增写在它后面，会被截胡跑不到，数值卡死不上涨，直接死循环；
因此 **迭代语句 i++/j++ 要写在循环体最前面**

### 分支结构
1. if分支：`if()` / `if()-else` / `if()-else if()-else`
2. switch分支：通过比较值来决定执行那条分支，与哪个case后的值匹配为true就执行哪个case的代码，遇到break就跳出switch分支；
如果全部case后的值与之匹配都是false，则执行default块的代码。
```java
switch（表达式）{
    case 值1：
        执行代码；
        break；
    case 值1：
        执行代码；
        break；
    ...
    default：
        执行代码；
}
```
3. 二者区别：功能上if远远强于switch；条件是区间时使用if分支；条件是单个值比较时，switch更适合，格式良好、性能较好、代码优雅。
4. switch注意事项：
表达式类型只能是 `byte、short、int、char`，**不支持 double、float、long、枚举、String**；
case的值不允许重复，只能是字面量，不能是变量；
一般都要加break，否则会出现**穿透现象**；
穿透性可简化代码：多个case执行相同逻辑时，可利用穿透只写一份代码。
```java
int month = 2;
switch (month){
    case 1:
    case 2:
    case 3:
        System.out.println("春季");
        break;
    case 4:
    case 5:
    case 6:
        System.out.prinln("夏季");
        break;
}
```

### Java程序运行内存原理
1. 运行一个java程序，主要看JVM中包含的**方法区、栈内存、堆内存**三部分内存区域。
2. 程序运行时，JVM 首先把磁盘上的.class字节码文件加载到方法区（元空间），存放类结构、方法信息、静态变量和常量；接着启动main方法，创建虚拟机栈帧开始逐行执行代码。
3. 基本数据类型变量，直接在栈内存中存放真实数值；
引用类型变量同样存在栈里，但只存堆内存的地址，不存对象本身；
根据这个地址，就能找到堆内存里实际的对象实例、成员变量和数组数据，实现栈引用指向堆实体的内存关联。
4. `int a=20;int[] arr = new int[3]` 执行原理：
a是变量，直接放在栈中，a变量中存储的数据就是20这个值；
new int[3]是创建一个数组对象，会在堆内存中开辟区域存储3个整数；
arr是变量，在栈中，arr中存储的是数组对象在堆内存中的地址值。
5. 注意：数组变量存`null`：表示栈中的数组引用变量，没有指向堆里任何数组对象。
可以直接打印该变量，但**不能访问数组长度、不能操作数组元素**，否则直接触发**空指针异常 NullPointerException**。

### 方法
- 方法：是一种语法结构，它可以把一段代码封装成一个功能，以便重复调用。
- 格式：
```java
修饰符 返回值类型 方法名（形参列表）{
    方法体代码（需要执行的功能代码）
    return 返回值；
}
```
1. 注意：方法申明了具体的返回值类型，内部必须使用return返回对应类型的数据；
形参列表可以多个可以没有，如果有多个必须用逗号隔开，且不能给初始化值。
2. 如果方法不需要返回数据，返回值类型必须设为`void`（无返回值申明），此时方法内部不可以使用return返回数据；
方法如果不需要接收数据，则不需要定义形参，且调用方法时也不可以传数据给方法；
没有参数且没有返回值类型申明的方法称为**无参数、无返回值的方法**，依次类推。
3. 使用方法时的常见问题：
a. 方法在类中的位置放前放后无所谓，但**一个方法不能定义在另一个方法里面**；
b. return语句的下面，不能编写代码，属于无效的代码，执行不到这儿；
c. 方法不调用就不会执行，调用方法时，传给方法的数据，必须严格匹配方法的参数情况；
d. 调用有返回值的方法，有3种方式：定义变量接收结果 / 直接输出调用 / 直接调用；
e. 调用无返回值的方法，只有1种方式：直接调用。
4. 设计方法的技巧：看方法是否需要接收数据处理、是否需要返回数据、明确方法业务功能

**方法重载**
-定义：一个类中出现多个方法的名称相同，但是它们的形参列表不同（个数、类型、顺序不同，不关注形参名称），那么这些方法就称为方法重载。
-应用：为处理一类业务提供多种解决方案
-提高代码复用
每个重载方法自己写一遍求和逻辑，代码大量重复
```java
public class SumDemo {
    // 求2个数和
    public static int sum(int a, int b){
        return a + b;
    }

    // 求3个数和
    public static int sum(int a, int b, int c){
        return a + b + c;
    }

    // 求4个数和
    public static int sum(int a, int b, int c, int d){
        return a + b + c + d;
    }

    public static void main(String[] args) {
        System.out.println(sum(1,2));
        System.out.println(sum(1,2,3));
        System.out.println(sum(1,2,3,4));
    }
}
```
只把核心逻辑写在参数最多的方法里，其他重载直接调用它
```java
public class SumDemo2 {
    // 2个数求和：复用3个数的方法，给默认值0
    public static int sum(int a, int b){
        return sum(a, b, 0);
    }

    // 3个数求和：复用4个数的方法，给默认值0
    public static int sum(int a, int b, int c){
        return sum(a, b, c, 0);
    }

    // 核心逻辑只写这1处！
    public static int sum(int a, int b, int c, int d){
        return a + b + c + d;
    }

    public static void main(String[] args) {
        System.out.println(sum(1,2));
        System.out.println(sum(1,2,3));
        System.out.println(sum(1,2,3,4));
    }
}
```
技巧：把完整、核心的业务逻辑，只写在参数最多的重载方法中，参数少的重载方法，不自己写逻辑，传默认值调用参数多的方法

### 数组反转
```java
package com.neau.study;
public class ArrayInvert {
    public static void main(String[] args) {
        int[] a={10,20,30,40,50};
        reverseArray(a);
        for(int j=0;j<a.length;j++){
            System.out.print(a[j]+" ");
        }
    }
    public static void reverseArray(int a[]) {
        for (int i = 0; i < a.length/2; i++) {
                int temp = a[i];
                a[i] = a[a.length-1-i];
                a[a.length-1-i] = temp;
            }
    }
}
```

### 代码技巧
1. 基础类型（int、double、boolean）、普通局部变量可放循环内；
new 出来的引用对象（Scanner、Random、集合、流、连接）**一律循环外只 new 一次**，避免频繁创建浪费内存。
2. 数组 / 集合遍历，永远用 `arr.length / list.size()`，不写死固定数字硬编码。
3. 变量先定义再使用，就近定义，不要把所有变量都堆在方法最上面。
4. 命名严格：小驼峰；方法名「动词 + 名词」如 reverseArray、printArray；见名知意，不用abc等无意义命名。
5. 尽量用 `int[] a` 标准写法，不写老式 `int a[]`。

### 面向对象编程（oop）

### 类和对象注意事项
1. 类名建议用英文单词，首字母大写，满足驼峰模式，且要有意义，如 Student、Car。
2. 类中定义的变量也称为**成员变量**（对象的属性），类中定义的方法也称为**成员方法**（对象的行为）。
3. 成员变量本身存在默认值，定义时一般不需要手动赋初始值。
4. 一个代码文件中，可以写多个class类，但**只能一个用public修饰**，且public修饰的类名必须和代码文件名一致。
5. 单个对象之间数据互不影响；**多个变量指向同一个对象时，数据会相互影响**。
6. 如果某个对象没有任何变量引用它，会成为垃圾对象；Java有**自动垃圾回收机制**，会自动清理垃圾对象。

### this
- this关键字：就是一个变量，用在方法中代表**当前对象的地址**，可以访问当前对象的成员变量和成员方法；哪个对象调用方法，this就指向哪个对象。
- 应用场景：区分成员变量和局部变量，解决名称冲突问题。
 ```java
public class Student {
    private String name; // 成员变量（对象属性）

    public void setName(String name) { // 局部变量（方法参数）
        // 这里两个 name 同名，不加 this 会优先找方法内的局部变量
        this.name = name; // this.name 表示当前对象的成员变量
    }
}
```

### 构造器
- 格式：方法名和类名完全一致，**无返回值类型**。
 ```java
public class Student {
    // 无参构造器
    public Student() {
    }
}
```
1. 格式：类名 对象名 = new 构造器；
2. 特点：创建对象时**自动调用**——`Student s = new Student();`
3. 作用：
无参数构造器——完成对象初始化，主要给成员变量（属性）赋值，并返回这个对象的地址，里面的数据都是默认值；
有参数构造器——完成对象初始化，主要给成员变量（属性）赋值，并返回这个对象的地址，同时为对象赋值。
4. 注意：类中不手动编写构造器时，Java会自动生成**默认无参构造器**；
一旦手动定义了有参构造器，默认无参构造器会消失，**建议手动补写无参构造器**，避免报错。

### 封装的设计规范：合理隐藏、合理暴露
- 面向对象三大特征：**封装、继承、多态**。
1. 合理隐藏，合理暴露：隐藏类内部敏感数据，只对外提供安全的访问方式，提供成套的getter和setter方法暴露取值和赋值。
2. 成员访问权限控制：
`public`（公开）：成员对外完全公开，可任意访问；
`private`（私有/隐藏）：成员仅类内部可访问，外部无法直接访问，用于隐藏核心数据。

### JavaBean（实体类）
- 实体类是一种特殊形式的类，用于封装数据，有 2 个强制规范：
1. 类中所有成员变量必须私有（private），并对外提供对应的`getXxx()`、`setXxx()`方法；
2. 类中必须包含**公共的无参构造器**。
- 应用：只负责数据存取，不处理业务逻辑，实现**数据层与业务层分离**。
- 补充：get/set方法、有参/无参构造器，都可以右键 `Generate` 快速生成。
 ```java
 public class Student {
    private String name;
    private double score;
    // 无参构造器
    public Student(){}
    // get/set方法
    public String getName(){return name;}
    public void setName(String name){this.name=name;}
    public double getScore(){return score;}
    public void setScore(double score){this.score=score;}
}
```

###局部变量和成员变量的区别
-成员变量	
类中位置：类里、方法外（属于对象）	
初始化值：有默认值（int=0、String=null、double=0.0）	
内存位置：堆内存（随对象创建）	
生命周期：对象存在则存在，对象销毁则消失	
作用域：整个类内所有方法都可使用	
-局部变量
类中位置：方法内 / 代码块内（临时使用）
初始化值：无默认值，使用前必须手动赋值，否则编译报错
内存位置：栈内存（随方法调用）
生命周期：方法执行期间存在，方法结束立即销毁
作用域：仅当前所在大括号 {} 内有效

###包：分门别类管理程序的，类似于文件夹，建包有利于程序的管理和维护，别人写好的程序通常都在别人的包里
在自己程序中调用其他包下的程序的注意事项：
1. 如果当前程序中，要调用自己所在包下的其他程序，可以直接调用(同一个包下的类，互相可以直接调用)；
2. 如果当前程序中，要调用其他包下的程序，则必须在当前程序中导包，才可以访问（包名+回车可以快速导包），导包格式:import包名.类名;
3. 如果当前程序中，要调用Java提供的程序，也需要先导包才可以使用，但是Java.lang包下的程序是不需要我们导包的，可以直接使用；
4. 如果当前程序中，要调用多个不同包下的程序，而这些程序名正好一样，此时默认只能导入一个程序，另一个程序必须带包名访问。
