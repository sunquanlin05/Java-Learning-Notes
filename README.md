1. **String 字符串**
   - 是不可变对象，每次修改都会生成新对象
   - `==` 比较内存地址，`equals()` 比较内容
   - 判空优先用 `str != null && !str.isEmpty()`，避免空指针异常

2. **for 循环**
   - for + i键 + 回车，快速生成 for 循环
   - 实际开发中，推荐把循环变量定义在 for 循环内部，避免变量名冲突
3. **if-else简化**
```java
   System.out.print(i == arr.length - 1 ? arr[i] : arr[i] + ", ");//用三元运算符简化
```
4. **while 循环**
```java
初始化语句;
while(循环条件)
{
    循环体语句;
    迭代语句;
}
```
   - 知道循环次数优先用 `for`
   - 不知道循环次数建议用 `while`
   - 功能上二者互通：能用 for 实现的都能用 while 实现，反之亦然

5. **do-while 循环**
```java
初始化语句;
do{
    循环体语句;
    迭代条件;
}while(循环条件);
```
   - 特点：**先执行一次，再判断条件**
   - 至少执行一次
   - 适用场景：抢票、点餐菜单选择、先展示再选择

6. **死循环 三种写法**
```java
for(;;){

}

while(true){

}

do{

```
   - 注意：同一文件只能跑一个死循环，其余要注释
   - 实际开发应用：服务器常驻运行、接口请求失败重试、定时轮询任务

7. **三种输出方式区别**
   - `print()`：只打印，**不换行**，适合同一行拼接多个内容
   - `println()`：打印完自动换行，日常最常用
   - `printf()`：格式化输出，可控制小数位数、占位符排版

8. **循环嵌套特点**
   - 外层循环每循环**一次**，内层循环**完整跑完一整轮**

9. **跳转关键词**
   - break：直接跳出整个循环，循环彻底结束，或者结束所在switch分支的执行
   - continue：跳过当前这一次，直接进入下一次循环，循环还继续跑；注意continue 一触发，迭代自增写在它后面，会被截胡跑不到，数值卡死不上涨，直接死循环，因此迭代语句 i++/j++ 要写在循环体最前面

10. **分支结构**
   - if分支：if()/if()-else/if()-else if()-else
   - switch分支：通过比较值来决定执行那条分支，与哪个case后的值匹配为true就执行哪个case的代码，遇到break就跳出switch分支，如果全部case后的值与之匹配都是false，则执行default块的代码
```java
switch（表达式）{
    case 值1：
        执行代码；
        break；
    case 值1：
        执行代码；
        break；
    ...
    default：
        执行代码；
}
```
   - 二者区别：功能上if远远强于switch，条件是区间时使用if分支；条件是单个值比较时，switch更适合，格式良好、性能较好、代码优雅
   - switch注意事项：表达式类型只能是byte、short、int、char，不支持double、float、long、枚举（enum）、String，且case的值不允许重复，只能是字面量，不能是变量，一般都要加break，否则会出现穿透现象（执行完当前分支会直接往下顺延执行后面所有分支，不再判断条件），但这种穿透性可以在某些情况下简化代码，比如当存在多个case分支的代码相同时，可以把相同的代码放到一个case块中，其他的case块都通过穿透性穿透到该case块执行代码即可
```java
int month = 2;
switch (month){
    case 1:
    case 2:
    case 3:
        System.out.println("春季");
        break;
    case 4:
    case 5:
    case 6:
        System.out.prinln("夏季");
        break;
}
```
   - 常用快捷键：ctrl+alt+t：包裹代码块/生成模板；ctrl+/：单行注释；shift+F6：批量重命名；shift+F10：运行当前程序

12. **数组定义与初始化**
   - 静态初始化数组格式：
     数据类型[] 数组名 = new 数据类型[]{元素1，元素2...}；
     简化格式：
     数据类型[] 数组名 = {元素1，元素2...}；
     //数据类型 数组名[]也可以，什么类型的数组只能存放什么类型的数据
   - 数组变量名存储的是数组在内存中的地址。数组是一种引用数据类型。
   - 动态初始化数组格式：
     数组类型[] 数组名 = new 数组类型[长度];//定义数组时先不存入具体的元素值，只确定数组存储的数据类型和数组的长度，先定义后赋值，注意静态初始化和动态初始化数组的写法是独立的

13. **Java程序运行内存原理**
   - 运行一个java程序，主要看JAVA中包含的方法区、栈内存、堆内存三部分内存区域
   - 程序运行时，JVM 首先把磁盘上的.class字节码文件加载到方法区（元空间），存放类结构、方法信息、静态变量和常量；接着启动main方法，创建虚拟机栈帧开始逐行执行代码
   - 基本数据类型变量，直接在栈内存中存放真实数值；引用类型变量同样存在栈里，但只存堆内存的地址，不存对象本身；根据这个地址，就能找到堆内存里实际的对象实例、成员变量和数组数据，实现栈引用指向堆实体的内存关联
   - int a=20;int[] arr = new int[3]这两行代码的执行原理是：
     a是变量，直接放在栈中，a变量中存储的数据就是20这个值
     new int[3]是创建一个数组对象，会在堆内存中开辟区域存储3个整数
     arr是变量，在栈中，arr中存储的是数组对象在堆内存中的地址值
   - 注意：数组变量存null：表示栈中的数组引用变量，没有指向堆里任何数组对象。可以直接打印该变量，但不能访问数组长度、不能操作数组元素，否则直接触发空指针异常 NullPointerException

15. **方法**
   - 方法：是一种语法结构，它可以把一段代码封装成一个功能，以便重复调用
   - 格式：
```java
修饰符 返回值类型 方法名（形参列表）{
    方法体代码（需要执行的功能代码）
    return 返回值；
}
```
   - 注意：方法申明了具体的返回值类型，内部必须使用return返回对应类型的数据，形参列表可以多个可以没有，如果有多个必须用逗号隔开，且不能给初始化值
   - 如果方法不需要返回数据，返回值类型必须设为void（无返回值申明），此时方法内部不可以使用return返回数据；方法如果不需要接收数据，则不需要定义形参，且调用方法时也不可以传数据给方法了；没有参数且没有返回值类型申明的方法称为无参数、无返回值的方法，依次类推
   - 使用方法时的常见问题：
     a.方法在类中的位置放前放后无所谓，但一个方法不能定义在另一个方法里面
     方法的返回值类型写void(无返回申明)时，方法内不能使用return返回数据，如果方法的返回值类型写了具体类型，方法内部则必须使用return返回对应类型的数据
     b.return语句的下面，不能编写代码，属于无效的代码，执行不到这儿
     c.方法不调用就不会执行，调用方法时，传给方法的数据，必须严格匹配方法的参数情况
     d.调用有返回值的方法，有3种方式:可以定义变量接收结果/直接输出调用（.sout+回车键可以快速输出）/直接调用;
     e.调用无返回值的方法，只有1种方式:直接调用
   - 设计方法的技巧：看方法是否需要接收数据进行处理/返回数据/方法要处理的业务（编程能力）

17. **数组反转**
```java
package com.neau.study;
public class ArrayInvert {
    public static void main(String[] args) {
        int[] a={10,20,30,40,50};
        reverseArray(a);
        for(int j=0;j<a.length;j++){
            System.out.print(a[j]+" ");
        }
    }
    public static void reverseArray(int a[]) {
        for (int i = 0; i < a.length/2; i++) {
                int temp = a[i];
                a[i] = a[a.length-1-i];
                a[a.length-1-i] = temp;
            }
    }
}
```

18. **代码技巧**
    - 基础类型（int、double、boolean）、普通局部变量可放循环内；new 出来的引用对象（Scanner、Random、集合、流、连接）一律循环外只 new 一次，避免频繁创建浪费内存
    - 数组 / 集合遍历，永远用 arr.length / list.size()，不写死 5、10 这种硬编码数字
    - 变量先定义再使用，就近定义，不要把所有变量都堆在方法最上面
    - 命名严格：小驼峰；方法名「动词 + 名词」reverseArray、printArray；见名知意，不用abc等无意义命名
    - 尽量用 int[] a 标准写法，不写老式 int a[]
      
19. **类和对象注意事项**
    - 类名建议用英文单词，首字母大写，满足驼峰模式，且要有意义，比如:Student、Car...
    - 类中定义的变量也称为成员变量(对象的属性)，类中定义的方法也称为成员方法(对象的行为)
    - 成员变量本身存在默认值，同学们在定义成员变量时一般来说不需要赋初始值(没有意义)
    - 一个代码文件中，可以写多个class类，但只能一个用public修饰，且public修饰的类名必须成为代码文件名
    - 对象与对象之间的数据不会相互影响，但多个变量指向同一个对象时就会相互影响了
    - 如果某个对象没有一个变量引用它，则该对象无法被操作了，该对象会成为所谓的垃圾对象，而java存在自动垃圾回收机制，会自动清除掉垃圾对象

20. **this**
    - this关键字：就是一个变量，可以用在方法，用来拿到当前对象；哪个对象调用方法，this就指向哪个对象，也就拿到哪个对象
    - 应用场景：区分成员变量和方法内部变量/局部变量，解决名称冲突问题
 ```java
public class Student {
    private String name; // 成员变量（对象属性）

    public void setName(String name) { // 局部变量（方法参数）
        // 这里两个 name 同名，不加 this 会优先找方法内的局部变量
        this.name = name; // this.name 表示当前对象的成员变量
    }
}
```

21. **构造器**
    - 格式：方法名和类名完全一致，无返回值类型
 ```java
      public class Student {
    // 无参构造器
    public Student() {
    }
}
```
    - 特点：创建对象时自动调用——Student s = new Student();
    - 作用：完成对象初始化，主要给对象的成员变量（属性）赋值
    - 注意：类中不手动编写构造器时，Java会自动生成一个默认无参构造器；一旦手动定义了有参构造器，Java不再自动生成无参构造器，建议手动补写无参构造器，避免报错
